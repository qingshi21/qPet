# qPet 开发计划


## 第1周：基础搭建

| 天数    | 后端                                  | 前端                             |
|-------|-------------------------------------|--------------------------------|
| Day 1 | 初始化 Spring Boot 项目，配置 MySQL 连接      | 初始化 Electron 项目，跑通 Hello World |
| Day 2 | 执行 schema.sql 建表，配置 MyBatis         | 学习 BrowserWindow，理解主/渲染进程      |
| Day 3 | 实现 UserMapper + PetsMapper 基础 CRUD  | 搭建启动器窗口 UI 骨架                  |
| Day 4 | 实现注册/登录接口（/api/auth/*）              | 启动器对接登录接口                      |
| Day 5 | 实现宠物列表/创建接口（/api/pets/*）            | 启动器展示宠物列表，支持创建                 |
| Day 6 | 集成 Qwen API，调通基础对话                  | 实现悬浮窗（透明、置顶、可拖动）               |
| Day 7 | **里程碑1**：注册→登录→创建宠物→能对话（Postman可测通） | 前后端联调登录→宠物列表→创建流程              |


## 第2周：核心功能

| 天数     | 后端                                | 前端                   |

| Day 12 | 实现 WebSocket 推送（主动冒泡定时任务）         | 前端 WebSocket 连接，接收推送 |
| Day 13 | 添加日志（SLF4J）和全局异常处理                | 双窗口通信（悬浮窗 ↔ 聊天窗口）    |
| Day 14 | **里程碑2**：对话+记忆+性格切换+主动冒泡全部跑通      | 前后端联调完整流程            |

## 扩展功能（进度快时考虑）

- 屏幕感知：截图 + AI 分析，让桌宠知道用户当前在做什么
- 自定义外观：换肤、换表情、换动画
- 多角色切换：同一宠物多套性格预设
- 语音输入输出：用 Web Audio API 实现语音对话
- 本地模型部署：接入 MiniCPM 等轻量模型
- 云端同步：多设备记忆共享
