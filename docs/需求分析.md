# AI桌宠项目需求分析

基于当前AI桌宠市场的发展状况（2025-2026年），以下是AI桌宠LumenAmi的需求分析。


## 一、市场背景

AI桌宠已从概念验证走向市场验证。

2024年AI外设的第一波浪潮（Humane AI Pin、Rabbit R1）试图替代手机，结果几乎全军覆没。2025-2026年的新方向做了一个关键修正：不再替代手机，而是在桌面上成为一个全新的存在。

当前市场已跑出多个成功案例：

- **Ropet**：售价约2000元，已售出近2万台，退货率仅4%，90天留存率超80%
- **LOOI**：Kickstarter众筹超51万美元，3578人支持，核心思路是“给手机一个身体”
- **StackChan**：开源AI桌宠，Kickstarter超目标45倍，4142人支持
- **可悠（Care U）**：商汤科技2026年4月正式发布，定位“家庭AI新成员”

**结论**：AI桌宠赛道已被验证有真实市场需求，且商汤科技本身正在布局这一方向。


## 二、目标用户

| 维度 | 描述 |
|---|---|
| **核心用户** | 大学生、刚工作的年轻人，长时间伏案使用电脑，希望即时的信息获取和问题解答 |
| **用户画像** | 一个人学习/办公，需要陪伴感；喜欢尝试新科技产品；对个性化定制有兴趣；需要高频互动和沟通 |
| **典型场景** | 写代码/写文档时，桌宠在屏幕角落陪伴；疲惫时可以聊两句；工作卡住时可以问问题 |
| **用户痛点** | 独自工作时的孤独感；需要情绪出口但不想麻烦他人；现有聊天机器人被动、缺乏“存在感”，互动不即时；不可自定义AI属性 |


## 三、竞品对比与差异化定位

| 产品 | 核心定位 | 差异化 |
|---|---|---|
| **Ropet** | 弱陪伴，刻意不做对话功能，主打物理抚摸和视觉反馈 | 做对话型陪伴，满足用户“想聊天”的需求 |
| **LOOI** | 借手机屏幕做脸，主打物理运动反馈 | 纯软件形态，降低用户门槛和成本 |
| **StackChan** | 完全开源，社区共创，硬件可改造 | 聚焦软件体验，无需用户动手改造 |
| **可悠** | 全场景智能体，跨设备协同，实体桌面机器人 | 纯桌面软件形态，更轻量、更易触达 |
| **Claw** | 生产力Agent，能操作电脑完成任务 | 陪伴型，不做操作，只做聊天和陪伴 |

**LumenAmi的差异化定位**：

> 一个轻量级、可自定义性格的AI陪伴桌宠——不是帮你干活的Agent，而是陪你聊天、有存在感的AI伙伴。


## 四、核心功能

### 4.1 必须功能

| 功能 | 优先级 | 说明 |
|---|---|---|
| 桌面悬浮显示 | P0 | 以可拖动小窗口形式悬浮在电脑桌面，始终置顶 |
| 基础对话 | P0 | 用户输入文字，AI以设定性格回复 |
| 性格自定义 | P0 | 用户可选择/设定桌宠的性格风格（如活泼、温柔、毒舌、治愈等） |
| 基础记忆 | P0 | 记住用户名字和关键偏好，在对话中自然带出 |
| 主动搭话 | P1 | 在用户一段时间未操作时，主动弹出气泡 |

### 4.2 扩展功能

| 功能 | 优先级 | 说明 |
|---|---|---|
| 屏幕感知 | P2 | 截取屏幕内容，让AI知道用户当前在做什么（如“你在写代码”），主动提供相关反馈 |
| 长期记忆升级 | P2 | 自动识别和存储重要信息，形成用户画像 |
| 自定义外观 | P2 | 用户可更换桌宠形象、颜色、表情风格 |

**优先级判断依据**：Ropet的创始人强调“不要让它说话”是刻意选择的定位，因为一旦说话，用户会期待它像智能音箱一样全知全能，反而容易出戏。但LumenAmi的差异化正在于“做对话型陪伴”，所以**对话和性格自定义是核心，屏幕感知是锦上添花**——先让用户能“用上”，再考虑“更酷”。


## 五、非功能性需求

| 维度 | 要求 |
|---|---|
| **性能** | 悬浮窗不卡顿，CPU/内存占用低 |
| **响应速度** | AI回复延迟不超过3秒（端到端） |
| **隐私** | 对话记录本地存储，或加密上传 |
| **易用性** | 开箱即用，无需复杂配置 |
| **兼容性** | 优先Windows，后续可考虑Mac |


## 六、市场前景

1. **市场已被验证**：Ropet、LOOI、StackChan等产品证明了用户愿意为AI桌宠付费
2. **商汤正在布局**：可悠的发布证明商汤科技看好这一方向，本项目可以视为“可悠”的软件先行版或纯软件探索
3. **差异化空间存在**：当前市面产品偏向“弱陪伴”或“硬件形态”，纯软件+强对话+自定义性格的桌宠仍是相对空白地带
4. **技术可行性高**：核心功能（对话+悬浮窗+记忆）均使用已有或能快速上手的技术，一个月内可交付可用版本


## 七、关键风险与应对

| 风险 | 应对策略 |
|---|---|
| 一个月内做不完屏幕感知功能 | 砍掉屏幕感知，先做对话+性格+记忆，这是完整的MVP |
| 用户不知道桌宠“能做什么” | 首次启动时提供简单的引导和性格选择界面 |
| AI回复太慢 | 使用响应速度快的API，考虑本地缓存 |
| 悬浮窗交互体验差 | 参考成熟桌面应用交互，保持简单直观 |
